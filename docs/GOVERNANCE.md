# IntellyWeave Governance

This document describes the governance model for the IntellyWeave project.

## Overview

IntellyWeave is an open-source OSINT analysis platform that welcomes community contributions. The project follows an open governance model, balancing maintainer oversight with community participation.

## Roles

### Project Lead

**Vero Dall'Aglio** ([@vericle](https://github.com/vericle)) serves as the project lead, responsible for:

- Overall project direction and vision
- Final decisions on architectural changes
- Release management
- Community growth and engagement

### Maintainers

Maintainers are trusted contributors with write access to the repository. They can:

- Review and merge pull requests
- Triage issues
- Participate in project decisions

Current maintainers are listed in [MAINTAINERS.md](MAINTAINERS.md).

### Contributors

Anyone who contributes code, documentation, bug reports, or other improvements. Contributors are the foundation of the project's growth.

## Decision Making

### Day-to-Day Decisions

- Bug fixes and documentation: Any maintainer can approve and merge
- Minor features: Requires maintainer review and approval
- Dependency updates: Managed via Renovate, reviewed by maintainers

### Significant Decisions

For larger changes (new features, breaking changes, architectural decisions):

1. Open a GitHub Issue or Discussion describing the proposed change
2. Allow time for community feedback
3. Maintainers discuss and reach consensus
4. Project lead makes final decision if consensus isn't reached

## Contributing

We welcome contributions from everyone. See [CONTRIBUTING.md](../CONTRIBUTING.md) for guidelines on:

- Creating feature branches
- Commit message conventions
- Testing requirements
- The pull request process

### Contributing to Upstream

IntellyWeave is built on Weaviate's Elysia framework. Improvements that benefit the broader community should be considered for upstream contribution. See [CONTRIBUTING.md](../CONTRIBUTING.md) for the upstream contribution workflow.

## Communication

- **GitHub Issues**: Bug reports and feature requests
- **GitHub Discussions**: Questions and general discussion
- **Pull Requests**: Code contributions
